# machine_of_creation

> .mecha lightweight pseudo-code format bridging contextextual codebase drift and semantic consistency

---

## What is `.mecha`?

The `machine_of_creation`(MOC) format inspired by Maps of Content and Zattelkasten. `machine_of_creation` or `mecha_god`:(extension:`.mecha`) is designed for optimal communication between AI agents and developers.

```
|  {3}human-readable
|   ^           v
|  {1}mecha_god{4}balance+ -> {5} output 
|   v           ^
|  {2}machine-parseable
```

---

## Core Syntax

```mech
{0}:block_identifier:
  [1]:property: "value"
  [2]:nested_structure:
    [a]:category_one:
      [i]: item
      [ii]: item -> {reference}
    [b]:category_two:
      [i]: item
  [3]:aliased_term(a:alias): "definition"
```

.mechaKey Patterns:.mecha
- `{n}:` → Top-level blocks
- `[n]:` → Numbered properties  
- `[a],[b],[c]...` → Lettered categories
- `[i],[ii],[iii]...` → Roman numeral items
- `(a:alias)` → Inline alias definition
- `->` → Links and references

---

## Example: Balance System

```mech
{1}:balance_system:
  [1]:concept: "weight-based feature balancing"
  [2]:components:
    [a]:minimal(a:miml):
      [i]: --:overhead_reduction
      [ii]: --:simplicity_focus
    [b]:feature(a:feat):
      [i]: ++:rich_cli_integration
      [ii]: ++:color_theming
  [3]:calculation:
    [tally(a:tly)]: {sum(++) - sum(--)} = balance(a:bal)
```

## Getting Started

```mech
{1}:your_first_mech:
  [1]:title: "My Project Config"
  [2]:agent(a:ai): "machine_of_creation"

    [ii]: [docs/](./docs/)

    [a]:balance(a:bal): feat(a:++) vs miml(a:--)
